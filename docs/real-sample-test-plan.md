# 真实手写样本测试方案

## 1. 文档概述
本文档用于定义 `handwriting-to-notion` 在 MVP 收口完成后的第一轮真实手写样本测试方案。  
目标不是做大规模自动化评测，而是通过一轮结构化的真实样本回归，判断当前 MVP 在真实输入下是否具备可验证价值，并为第二轮优化方向提供依据。

本文档默认基于以下前提：
- MVP 主链路已闭环
- 第一轮收口任务已基本完成
- 当前项目已进入“真实样本测试准备与执行阶段”

## 2. 测试目标
本轮样本测试的核心目标如下：

1. 验证 OCR 原文在真实手写输入下是否具备基本可读性
2. 验证 Structured Notes 是否具备明显整理价值
3. 验证 To-do List 是否能提供可用的行动项提取
4. 验证用户是否会愿意把结果复制到 Notion
5. 判断当前主要瓶颈位于：
   - 预处理
   - OCR
   - 结构化
   - 页面表达 / 使用理解

## 3. 测试范围
本轮测试覆盖：
- 单张手写图片上传
- OCR 原文输出
- Structured Notes 输出
- To-do List 输出
- warning / error 提示体验
- 复制前的结果可用性判断

## 4. 非范围
本轮默认不覆盖：
- Notion API 直连
- 多图上传 / 合并
- 历史记录
- 账号系统
- 编辑器能力
- 阶段 2 / 阶段 3 能力验证
- 自动化 benchmark 体系

## 5. 样本选择原则
样本应尽量覆盖真实使用场景，而不是只选“最好识别”的样本。  
建议同时覆盖：
- 好样本
- 一般样本
- 差样本

目标是用有限样本快速识别当前 MVP 的能力边界。

## 6. 建议样本类型
建议第一轮准备 **20 ~ 30 张** 样本，尽量覆盖以下类型：

### 6.1 按场景划分
- 课堂笔记
- 会议记录
- 便利贴 / sticky note
- 草稿纸
- 日常待办页
- 研究 / 读书摘记页

### 6.2 按清晰度划分
- 清晰手写
- 一般清晰度
- 潦草手写

### 6.3 按结构划分
- 简单线性笔记
- 有项目符号 / 列表
- 有明显任务项
- 无明显结构文本
- 有混合短句 / 关键词

### 6.4 按输入复杂度划分
- 单语内容
- 中英混写
- 简单段落
- 多行碎片信息
- 有删改、圈画、箭头等轻度噪音

## 7. 建议样本分布
可采用以下分布作为第一轮参考：

- 6 张清晰笔记页
- 5 张一般清晰度笔记页
- 4 张潦草手写页
- 4 张会议/课堂待办型页面
- 3 张便利贴 / 碎片型页面
- 3 张中英混写页面

总量约 25 张。

## 8. 测试执行方式
每张样本建议走同一套流程：

```text
上传图片
→ 查看 OCR Plain Text
→ 查看 Structured Notes
→ 查看 To-do List
→ 查看 warning / error
→ 判断是否愿意复制到 Notion
→ 记录问题与结论
```

## 9. 评估维度
每张样本至少记录以下 4 个核心维度：

### 9.1 OCR 可读性
问题：
- OCR 原文是否基本可读？
- 是否足以作为人工校对基础？

建议评分：
- 2 = 可读，基本能理解
- 1 = 部分可读，但需要较多猜测
- 0 = 基本不可读

### 9.2 Structured Notes 可用性
问题：
- 是否比 Plain Text 更容易贴进 Notion？
- 是否体现出整理价值？

建议评分：
- 2 = 明显有用
- 1 = 有一点帮助，但价值一般
- 0 = 没帮助或容易误导

### 9.3 To-do List 可用性
问题：
- 是否能提取出可信的行动项？
- 是否适合直接作为 checklist 使用？

建议评分：
- 2 = 明显有用
- 1 = 部分有用，需要人工修改
- 0 = 不可靠或无价值

### 9.4 复制意愿
问题：
- 如果你是目标用户，是否愿意复制其中一种结果进 Notion？

建议评分：
- 2 = 愿意直接复制
- 1 = 需要人工修改后才愿意复制
- 0 = 不愿意复制

## 10. 补充记录项
除评分外，每张样本还建议记录：
- 样本编号
- 样本类型
- 图片清晰度
- 是否出现 warning
- 主要失败原因
- 哪一种输出最可用

## 11. 建议记录模板
可用如下表头记录：

| 样本ID | 场景类型 | 清晰度 | OCR可读性(0-2) | Structured Notes(0-2) | To-do List(0-2) | 复制意愿(0-2) | warning | 最可用输出 | 主要问题 |
|---|---|---:|---:|---:|---:|---:|---|---|---|

## 12. 结果判断方法
测试完成后，应优先看整体趋势，而不是单张好坏。

### 12.1 如果 OCR 可读性普遍偏低
说明当前主要瓶颈在：
- 图片预处理
- OCR provider

下一轮优先方向：
- 优化 preprocess
- 优化 OCR

### 12.2 如果 OCR 可读，但 Structured Notes / To-do List 普遍偏弱
说明当前主要瓶颈在：
- 结构化逻辑
- 结果组织方式

下一轮优先方向：
- 增强 restructure provider
- 评估是否引入 LLM 结构化

### 12.3 如果结果基本可用，但复制意愿仍低
说明主要问题可能在：
- 产品表达不够强
- 输出格式不够贴合 Notion
- 用户不确定该相信哪种结果

下一轮优先方向：
- 优化结果说明
- 优化 copy format
- 优化 warning / guidance

## 13. 验收标准
本轮样本测试完成的标准为：
- 已准备并执行一轮真实样本测试
- 已完成每张样本的结构化记录
- 已能明确归纳当前主要瓶颈
- 已能支持第二轮优化方向决策

## 14. 输出物
本轮测试建议至少产出：
1. 一份样本记录表
2. 一份测试结论摘要
3. 一份第二轮优化优先级建议
