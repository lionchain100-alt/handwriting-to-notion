## 1. 文档概述
本文档定义 `handwriting-to-notion` 第一阶段 MVP 的产品需求，用于快速验证“手写笔记 → Notion-ready 输出”这一核心价值是否成立。

## 2. 背景与目标

### 2.1 背景
用户在课堂、会议、便利贴、草稿纸等场景下会产生大量手写内容。用户真正需要的不是单纯 OCR，而是能快速整理并粘贴到 Notion 的结构化结果。

### 2.2 目标
- 验证用户是否理解并认可产品价值
- 验证三种输出模式是否具备吸引力
- 验证用户是否愿意把结果复制到 Notion
- 用最低成本快速上线可用版本

## 3. 范围

### 3.1 本阶段范围
- 上传单张手写图片
- 展示图片预览
- OCR 提取原文
- 输出纯文本、结构化笔记、待办清单
- 支持复制为 Markdown / bullet list / checklist
- 提供基础处理状态和错误提示

## 4. 非范围
- 不做账号系统
- 不做 Notion API 直连
- 不做多图合并
- 不做复杂场景模板
- 不做深度版面理解
- 不做人工编辑器
- 不做历史记录和项目管理能力

## 5. 用户流程
```text
进入页面
→ 上传手写图片
→ 系统处理图片
→ OCR 提取原文
→ LLM 整理输出
→ 用户切换查看三种结果
→ 用户复制到 Notion
```

## 6. 功能需求

### 6.1 上传与预览
- 用户可上传单张图片
- 页面展示图片预览
- 用户可重新选择图片
- 对不支持的输入格式进行提示

### 6.2 识别与输出
- 系统对图片进行基础预处理
- 系统提取 OCR 原文
- 系统输出三种结果：
  - Plain Text
  - Structured Notes
  - To-do List

### 6.3 复制能力
- 支持复制纯文本
- 支持复制为 Markdown
- 支持复制为 bullet list
- 支持复制为 checklist

### 6.4 状态与提示
- 显示处理中、完成、失败状态
- 对识别可能不准确的结果提供 warning
- 发生错误时提供可理解提示

## 7. 技术方案
- 图像预处理：`sharp`
- OCR baseline：`Tesseract.js`
- 结构化输出：`LLM`
- 承接流程：Next.js API Route
- 工程实现遵循 `CLAW.md`：types → services → API → UI

## 8. 验收标准
- 用户能完成从上传到复制结果的完整流程
- 页面价值表达清晰
- Structured Notes 和 To-do List 有明显使用价值
- 能收集到真实失败样本和真实手写输入数据
- 代码结构支持后续 OCR provider 升级

## 9. 风险与依赖

### 9.1 风险
- Tesseract.js 对复杂手写效果有限
- 差图输入会显著影响识别结果
- 结构化层可能出现幻觉或过度整理

### 9.2 应对策略
- 保留 Plain Text 作为原文兜底
- 先验证产品价值，不追求第一阶段最强识别效果
- 加强 warning 提示，降低用户误解

### 9.3 依赖
- Tesseract.js OCR 能力
- LLM 输出质量
- 用户真实输入样本反馈
